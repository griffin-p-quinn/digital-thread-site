NX MCP in Practice

What We Have Actually Built With NX MCP

This work is not just about tool plumbing. The repo now carries a real design-generation story: governed NX remoting, benchmark-grade geometry builds, and a growing family of generated concepts spanning vehicles, aircraft, ships, and underwater systems.

Benchmarks and Capability Proof
verified

From MCP calls to real generated geometry

The workspace state shows a progression from primitive body creation to associative rule graphs, live FEM, reloadable NX handler overrides, ship-design features, and full benchmark builds. The important point is not that NX MCP can call NX once. It is that it can drive repeatable, multi-step geometry stories end to end.

Tool Surface
68 NX-facing tools across geometry, rules, simulation, ship features, and remoting workflows.

GX-3 Proof Points

Operational Reality
Benchmarks Create Fresh Parts On Purpose
Most of the named builders in this repo are full benchmark scripts, so they intentionally create a new part and then drive an entire geometry story from start to finish rather than editing an existing model in place.
CreateNewPart benchmark flow
Existing Parts Are Already Supported
The platform is not limited to greenfield generation. The facade already exposes part-open and part-save operations, so NX MCP can work from an existing part when the workflow calls for that instead of a benchmark reset.
OpenPart · SavePart
Current Save Default Is Still Rough
Today the default part-creation path lands under the user profile, which is useful for quick isolated runs but not a clean long-term project convention. That is a current implementation choice, not a product claim.
%USERPROFILE% default path

About this site

Digital Fabric

A living knowledge base from the Emerging Technologies team at Siemens DISW. These presentations document real architecture decisions, system integration patterns, and lessons learned at the intersection of AI and industrial software.

Purpose
Document and share architecture patterns for AI + CAD/CAM/CAE/PLM integration at production scale. Each study is a standalone, interactive reference.
Approach
Informed by real implementation. Reference pages are grounded in live NX, Teamcenter, or Mendix work; experimental pages are explicitly labeled so the presentation style does not get mistaken for the validation level.
Audience
Solution architects, technology leads, and engineering teams evaluating AI-driven automation of industrial design and manufacturing workflows.
Cadence
New presentations are added as the team completes architecture studies. Reference decks pass internal review before going live; experience-lab pages stay clearly labeled when they are testing a new storytelling format.
