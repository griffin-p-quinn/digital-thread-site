System overview

Teamcenter MCP — agentic PLM integration

A reference MCP architecture that gives AI agents structured, governed access to Teamcenter. 60+ tools sit behind runtime mutation gates and deterministic session control for search, BOM, change management, 3D/JT, vendor intelligence, OFAC screening, and program delivery.

Architecture principle

SOA bridge with mutation safety

Wraps Teamcenter SOA services as typed MCP tools with schema validation and runtime policy enforcement. Mutations require explicit TC_MUTATION_MODE activation. Read-only tools are always available.

Transport
Streamable HTTP :8088/mcp + SSE push. Stdio for local dev.
Auth
SOA credentials + CSRF bootstrap. Service accounts for headless.
Mutation gate
TC_MUTATION_MODE controls access. Runtime changeable via admin tools.
Session
Deterministic guard — explicit tc_login required. No implicit sessions.

Canonical flow
Beat: write operations are allowed only when policy says so, and every change must end in deterministic verification rather than hope.

Read this top to bottom. The amber bar is the mutation gate, the left column is the recommended execution sequence, and the right callouts show optional adjacent actions or anti-patterns.

off
Read-only. All mutations blocked. Safe for demos.
managed
Create/relate OK. Workflow submit blocked.
full_write

Preferred flow: discover types, create with submitToWorkflow=true, then verify outcome. Anti-pattern: do not preflight with tc_get_workflow_templates unless explicitly needed.
