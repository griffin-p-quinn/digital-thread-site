01 · Problem

A one-off Gemini response is hard to come back to

General video analysis becomes difficult to compare when the input, provider, report, and prior runs are scattered across uploads and chat sessions.

The project asks how a local tool can make that work reproducible without pretending to be a full video production suite. Its job is narrower: organize a request and retain its structured evidence.

02 · My contribution

I wrapped the model call in a small desktop app

I packaged the analysis flow as a desktop workspace instead of leaving it as a disposable prompt-and-response exchange.

  • Kept input selection local. The user chooses a video from the desktop workflow and sees its working context.
  • Made the Gemini call explicit. The analysis request is sent to Gemini through the configured desktop workflow rather than hiding the model path.
  • Retained structured output. Scene reports and run history remain available for inspection after an individual request completes.
03 · Proof

The real app covers upload, report generation, and history

The source record and desktop capture document local selection, provider analysis, structured scene reporting, and retained history as one packaged workflow.

  • Packaged surface: the project is recorded as a desktop application, not a web-only concept.
  • Structured result: the primary output is a scene report rather than an unorganized chat transcript.
  • Persistent working state: run history is part of the workspace and remains visible beside the current report.
  • Source definition: the project README documents upload, provider analysis, report structure, and history.

Portfolio boundary: the browser demo is the real exported application frontend wired to a local sample API. It does not upload a video or call Gemini, but the interface, report views, history behavior, and interaction flow come from the working app.

04 · Result

Each run is saved instead of disappearing into a chat

The result is a repeatable local analysis loop: choose a video, choose the configured provider path, request a report, and retain the outcome with its history.

No claim is made about model accuracy or time saved. The demonstrated consequence is organizational: each run has an inspectable place and a structured record.

Run model

Video → Gemini → report → SQLite

The tool’s architecture is deliberately small. The product value sits in preserving the relationship between a local input, the configured request path, and the retained output.
