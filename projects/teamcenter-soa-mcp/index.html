01 · Governed access

Problem

An agent can only be useful in PLM if it works through real sessions and real operations.

Reading product data, editing objects, and performing broad writes are materially different authorities; flattening them into one tool permission would make the integration hard to audit and harder to trust. The project tests whether an LLM can operate Teamcenter through SOA while those levels remain explicit at runtime.

02 · Gateway design

My contribution

I built the Python/FastMCP gateway around authenticated Teamcenter sessions.

I registered the 77-tool surface and supported both local and Eureka-discovered connection modes. I added resources and prompts around the tool layer and enforced separate read, edit, and full-write gates at runtime.

04 · Outcome

Result

The gateway makes Teamcenter operations agent-addressable without hiding the enterprise session.

It does not collapse all access into a single permission. The result is a broad MCP surface whose authority can be inspected and constrained at the point of use.
