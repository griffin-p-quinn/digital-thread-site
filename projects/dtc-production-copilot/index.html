01 · Problem

The alert, plant view, and reasoning were in different places

A throughput alert is not useful by itself. An operator still has to connect the affected station, the plant layout, the available MES evidence, and a credible recovery path.

The project asks whether that investigation can happen in one legible surface without turning the assistant into a black box. The recommendation should stay attached to the station state and to the tools used to produce it.

02 · My contribution

I put the plant state and the co-pilot in one workflow

I composed the 3D plant, production dashboard, station views, alert flow, and co-pilot into one React experience.

Role of generative AI. The co-pilot interprets the visible manufacturing state and calls explicit diagnosis, recovery, and simulation tools; I designed that tool sequence and kept its evidence visible.

  • Connected spatial and operating context. The line view and station state tell the same production story rather than living in separate demos.
  • Made the assistant inspectable. The diagnosis, recovery, and simulation stages expose the tool sequence behind the recommendation.
  • Supported two operating modes. A scripted path keeps the public build repeatable; the source also defines a live diagnostic path for a configured backend.
03 · Proof

The browser build shows the whole scripted path

The linked artifact is the real static React build running in scripted mode, not a replacement illustration.

  • Browser-safe build: launch the alert, inspect the 3D line, and follow diagnosis → recovery → simulation without a backend.
  • Private source record · frontend composition: app/frontend/src/App.tsx composes the production view, station state, alert flow, and co-pilot panel.
  • Private source record · tool contracts: app/backend/dtc_copilot/tools.py defines the diagnostic, recovery, and simulation interfaces used by the assistant flow.

Public-demo boundary: the deployed case uses scripted, browser-safe data. It does not connect to a live MES, plant, model provider, or credential store.

04 · Result

An operator can see the shortfall and how the recommendation was reached

The finished prototype turns a production shortfall into a traceable investigation: where the problem is, what the station reports, what the assistant checked, and what recovery path it proposes.

No time or cost reduction is claimed. The demonstrated result is a more inspectable way to connect spatial context, operating state, and agent reasoning in one workflow.

System view

Alert → diagnosis → recovery

The interface preserves a straight evidence chain. The public build substitutes a deterministic scenario at the system boundary while leaving the product interaction intact.
